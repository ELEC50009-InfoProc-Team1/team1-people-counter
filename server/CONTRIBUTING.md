# Contributing to the server

TL;DR: Use VSCode, don't push to main, use Conventional Commits, take repo issues up with [@georgerob801](https://github.com/georgerob801).

Please for the love of god no vibe coding.

## Visual Studio Code

This respository has a Visual Studio Code workspace set up - after cloning the repository, open its folder and install any recommended extensions when prompted.

## Git

Because Github is Github, the `main` branch cannot be protected until this repository becomes public. As such, while I can't stop you, *please* don't push to `main`. Please open a pull request with changes when you want to do something and wait for it to be reviewed before merging it to `main`. much thank

Generally follow the format as described by [https://nvie.com/posts/a-successful-git-branching-model/](https://nvie.com/posts/a-successful-git-branching-model/).

## Commit messages

After setting the workspace up in Visual Studio Code, you should have Conventional Commits installed - please use that to write your commit messages. It can be found on the Source Control tab as the circle next to the tick on the "changes" window.
